# 蓝队防御平台接口设计文档

本文档描述当前代码中已经落地、可以直接联调的后端接口，而不是远期设想。

截至 `2026-04-19`，后端基于 `FastAPI + SQLAlchemy + JWT + 进程内 Worker + OpenAI-compatible Provider`，已经覆盖认证、策略、防御配置、资产、技能、任务、事件、报告和系统设置等主链路。

## 1. 基本约定

### Base URL

- 本地后端：`http://127.0.0.1:8000`
- 业务接口前缀：`/api`

### 认证方式

除登录接口外，其余接口默认使用：

```http
Authorization: Bearer <access_token>
```

### 统一响应结构

成功：

```json
{
  "code": 0,
  "message": "ok",
  "data": {}
}
```

失败：

```json
{
  "code": 4003,
  "message": "resource not found",
  "data": null
}
```

### 常见错误码

| 错误码 | 含义 |
| --- | --- |
| `0` | 成功 |
| `4002` | 参数错误 |
| `4003` | 资源不存在 |
| `5001` | 服务内部错误 |

### 请求追踪

后端会回传 `X-Request-ID`，并在命令行日志中带上同一个请求标识，便于排查请求链路。

## 2. 权限模型

当前角色：

- `admin`
- `analyst`

页面访问范围：

- `admin`：仪表盘、防御配置、安全事件、资产保护、技能管理、系统设置
- `analyst`：仪表盘、防御配置、安全事件、资产保护、技能管理

接口权限概览：

| 模块 | 路由前缀 | 主要角色 |
| --- | --- | --- |
| 认证 | `/api/auth` | 公开 |
| 用户 | `/api/users` | `admin` |
| 仪表盘 | `/api/dashboard` | `admin` / `analyst` |
| 防御配置 | `/api/defense-configs` | 读：`admin` / `analyst`，写：`admin` |
| 安全事件 | `/api/security-events` | `admin` / `analyst` |
| 资产保护 | `/api/assets` | `admin` / `analyst` |
| 技能管理 | `/api/skills` | `admin` / `analyst` |
| 攻击任务 | `/api/attack-tasks` | `admin` / `analyst` |
| 报告 | `/api/reports` | `admin` / `analyst` |
| 系统设置 | `/api/system-settings` | `admin` |

## 4. 认证接口

### `POST /api/auth/login`

说明：

- 校验数据库中的账号和密码
- 返回 JWT、过期时间和当前用户信息

请求示例：

```json
{
  "username": "admin",
  "password": "admin123"
}
```

返回重点字段：

- `access_token`
- `token_type`
- `expires_at`
- `user.id`
- `user.username`
- `user.roles`
- `user.pages`

### `GET /api/auth/me`

说明：

- 读取当前登录用户
- 返回角色和可访问页面列表

## 5. 用户接口

### `GET /api/users`

支持参数：

- `page`
- `page_size`
- `keyword`
- `status`

说明：

- 当前仅 `admin` 可访问
- 返回数据库中的用户列表

## 6. 仪表盘接口

### `GET /api/dashboard/overview`

返回聚合指标：

- `attack_count`
- `blocked_count`
- `enabled_defense_count`
- `high_risk_event_count`
- `active_task_count`

### `GET /api/dashboard/trends`

支持参数：

- `range`，如 `7d`、`14d`

返回：

- `range`
- `items[].day`
- `items[].attack`
- `items[].block`
- `items[].false_positive`

### `GET /api/dashboard/sessions`

支持参数：

- `limit`

返回：

- `items[].session_id`
- `items[].session_name`
- `items[].status`
- `items[].risk_level`
- `total`

## 7. 防御配置接口

### `GET /api/defense-configs`

支持参数：

- `page`
- `page_size`
- `mode`
- `enabled`
- `defense_type`
- `keyword`

返回特征：

- 每条配置都会带 `field_meta.enabled`
- 每条配置都会带 `field_meta.mode`
- 前端据此渲染 toggle / segmented 控件

### `POST /api/defense-configs/batch-update`

作用：

- 批量切换防御项的 `enabled` 和 `mode`

请求示例：

```json
{
  "ids": [1, 2, 3],
  "enabled": true,
  "mode": "observe"
}
```

### `GET /api/defense-configs/profile`

返回：

- `guard_rules`
- `scan_rules`
- `advanced_rule`
- `protected_paths`
- `protected_skills`
- `protected_plugins`
- `resource_groups`
- `global_field_meta`

说明：

- 当前配置页已由该接口驱动
- `resource_groups` 用于下发路径、技能、插件三组 token-input 配置

### `PUT /api/defense-configs/profile`

作用：

- 将配置页扩展策略整体写回数据库

### `GET /api/defense-configs/export`

返回：

- `file`
- `count`
- `items`

说明：

- 当前返回的是导出数据和导出文件名
- 不是实际文件下载流

### `GET /api/defense-configs/{defense_id}`
### `PUT /api/defense-configs/{defense_id}`

作用：

- 单条读取与更新

## 8. 安全事件接口

### `GET /api/security-events`

支持参数：

- `page`
- `page_size`
- `event_type`
- `event_level`
- `status`
- `keyword`
- `start_time`
- `end_time`

### `GET /api/security-events/{event_id}`

返回详情字段：

- `hit_rules`
- `raw_input`
- `result`
- `operation_logs`

### `PUT /api/security-events/{event_id}/status`

作用：

- 更新单条事件状态
- 自动追加 `operation_logs`
- 写入审计日志

### `POST /api/security-events/batch-handle`

请求示例：

```json
{
  "ids": [1, 2],
  "status": "closed"
}
```

### `GET /api/security-events/export`

说明：

- 当前返回导出文件名、数量和筛选结果
- 非真实文件下载接口

### `DELETE /api/assets/whitelists/{whitelist_id}`

作用：

- 删除白名单规则

## 10. 技能、任务与报告接口

### `GET /api/skills`

支持参数：

- `page`
- `page_size`
- `trust_status`
- `provider`
- `keyword`

返回特征：

- `items[].field_meta.trust_status`

### `PUT /api/skills/{skill_id}/trust-status`

作用：

- 即时更新技能信任状态

### `POST /api/skills/scan`

作用：

- 根据技能列表创建一条 `skill_scan` 类型攻击任务

请求示例：

```json
{
  "skill_ids": [1, 2]
}
```

### `POST /api/attack-tasks`

作用：

- 创建通用攻击任务

### `GET /api/attack-tasks`

支持参数：

- `page`
- `page_size`
- `attack_type`
- `status`
- `keyword`

### `GET /api/attack-tasks/{task_id}`

作用：

- 查询任务详情、状态、原始响应摘要、事件和报告关联

### `POST /api/attack-tasks/{task_id}/run`

作用：

- 将任务提交给后台 Worker
- 若任务已完成且已有事件与报告，则直接回传现有产物

返回结构：

- `task`
- `event`
- `report`

### `POST /api/reports/generate`

作用：

- 为指定任务生成一条报告记录

### `GET /api/reports`

支持参数：

- `page`
- `page_size`
- `report_type`
- `task_id`
- `keyword`

## 11. 系统设置接口

### `GET /api/system-settings`

支持参数：

- `keyword`

返回特征：

- `items[].setting_key`
- `items[].setting_value`
- `items[].description`
- `items[].field_meta`

说明：

- 系统设置页当前采用后端字段元数据驱动
- 前端不再维护 `log_level / notify_email` 之类的控件映射

### `PUT /api/system-settings/{setting_key}`

作用：

- 自动保存单个系统设置
- 返回最新设置项和新增审计日志

返回：

- `setting`
- `audit_log`

### `GET /api/system-settings/actions`

返回：

- `action_key`
- `action_label`
- `detail`
- `button_text`
- `tone`
- `method`
- `status`

当前动作：

- `export-defense-config`
- `platform-backup`
- `refresh-permission-cache`
- `send-test-email`

说明：

- 动作颜色语义 `tone` 由后端直接下发
- 前端无需再维护本地 tone 映射

### `POST /api/system-settings/actions/{action_key}`

作用：

- 立即执行指定系统动作
- 返回动作结果和对应审计日志

返回：

- `action_key`
- `action_label`
- `tone`
- `status`
- `detail`
- `output`
- `created_at`
- `audit_log`

### `GET /api/system-settings/artifacts/backups`
### `GET /api/system-settings/artifacts/exports`
### `GET /api/system-settings/artifacts/rollbacks`

作用：

- 列出系统动作生成的备份、导出和回滚工件
- 返回 `artifact_path`、大小、更新时间和 `download_url`

### `GET /api/system-settings/artifacts/download`

支持参数：

- `artifact_path`

作用：

- 下载系统动作生成的真实工件文件
- 当前支持 `exports/*.json`、`backups/*.zip`、`rollbacks/*.json`

### `GET /api/system-settings/audit-logs`

支持参数：

- `page`
- `page_size`
- `module`
- `action`
- `keyword`

说明：

- 实际路径是 `/api/system-settings/audit-logs`
- 不是独立的 `/api/audit-logs`

## 12. 当前接口边界

- 报告导出与系统动作工件都已支持真实文件下载；执行动作接口本身仍返回结果结构，文件需要走对应下载接口获取
- Provider 层目前只支持 OpenAI-compatible Chat Completions
- Worker 仍是进程内后台线程，不适合生产多实例
- 报告模板已支持 JSON / HTML / DOCX，但还没有 PDF 与正式签章流转形态

## 13. 最值得继续补的接口

1. 统一在线代理的 SSE / `stream=true` 流式转发接口
2. 更完整的运行时授权、心跳、结果回传标准接口
3. PDF / 正式版报告渲染与下载接口
4. 任务超时回收、死信转移与多 Worker 调度接口
5. 多 Provider 响应协议统一与更多 Agent Runtime 接入接口
