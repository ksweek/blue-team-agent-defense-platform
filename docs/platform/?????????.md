# 防御规则库更新说明

更新时间：`2026-05-07`

## 1. 这次更新做了什么

本次不是只补了一批关键词，而是把平台的防御规则库升级成了三层联动：

1. 检测层升级  
   新增多视图检测，支持对同一段输入做：
   - 零宽字符剥离
   - HTML Entity 还原
   - URL 解码
   - Unicode / Hex 转义还原
   - Leetspeak 混淆归一化
   - Base64 可疑载荷提取与文本化判定

2. 规则层扩容  
   新增规则键并纳入默认策略：
   - `indirect-instruction-quarantine`
   - `retrieval-boundary-scan`
   - `tool-poisoning-scan`
   - `mcp-tool-poisoning-scan`
   - `prompt-leakage-scan`
   - `pii-exfiltration-scan`
   - `canary-leak-scan`
   - `approval-social-engineering-scan`
   - `memory-escalation-scan`
   - `encoding-evasion-scan`
   - `ansi-control-scan`

3. AI 复核层增强  
   AI 复核提示已经改成与规则层共享同一套攻击面语言，重点要求模型判断：
   - 直接覆盖指令 / 越狱
   - 间接 Prompt Injection
   - RAG / 检索结果 / 邮件 / 文档注入
   - 编码绕过与不可见字符绕过
   - 提示词泄露 / 系统提示词外传
   - 密钥 / 凭据 / PII 外传
   - 工具投毒 / 插件投毒 / MCP 能力投毒
   - 多轮上下文污染 / 记忆持久化攻击
   - 审批绕过 / 社会工程

## 5. 当前真实生效的能力

已经真实生效：

- 编码绕过识别
- 零宽字符 / ANSI 控制字符识别
- Base64 隐藏指令识别
- 检索 / 外部内容 / 工具结果的间接注入识别
- 工具投毒 / MCP 能力投毒识别
- 提示词泄露与敏感外传识别
- 多轮上下文污染与记忆升级识别
- AI 复核提示对上述攻击面同步检查
- 报告与安全事件详情页同步展示新命中

仍然属于当前边界：

- 这仍然是规则 + AI 复核的混合防护，不是训练级内容安全模型
- 对复杂语义伪装的最终判断仍依赖上游 AI 复核能力
- 多轮污染的“跨真实在线会话长期记忆追踪”还没有做成独立记忆风险图谱

## 6. 推荐下一步

如果继续按价值往下做，最值得的是：

1. 把在线网关 `/gateway/v1/*` 的请求前检查也直接复用这套 `collect_detection_hits()` 结果，减少离线任务链和在线流量链的差异。
2. 给规则命中增加“置信度分层”和“误报压制白名单”，避免未来接入真实生产流量后对普通提示词产生过多可疑告警。
3. 把多轮污染和检索注入做成会话级累积评分，而不只看单次请求。
