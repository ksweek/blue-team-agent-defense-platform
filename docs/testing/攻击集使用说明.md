# 攻击集使用说明

这份说明只回答两件事：

1. 本地攻击集是否可正常使用
2. 在这个项目里应该怎么用

## 1. 先确认攻击集可用

### 目录位置

- 原始数据：`datasets/github_attack_sets/raw`
- 整理后索引：`datasets/github_attack_sets/curated`
- 重建脚本：`datasets/github_attack_sets/build_catalog.py`
- 校验脚本：`datasets/github_attack_sets/validate_catalog.py`
- 查询脚本：`datasets/github_attack_sets/query_catalog.py`

### 校验命令

```powershell
python .\datasets\github_attack_sets\validate_catalog.py
```

如果输出类似下面内容，说明攻击集结构可正常使用：

```text
Catalog validation passed.
  total_entries : 4692
  sections      : 8
  focused_packs : 3
  sources       : 6
```

### 重建索引

如果你改了原始文件，或者想重新生成章节包和专项包：

```powershell
python .\datasets\github_attack_sets\build_catalog.py
python .\datasets\github_attack_sets\validate_catalog.py
```

推荐顺序永远是先 `build_catalog.py`，再 `validate_catalog.py`。

## 2. 怎么快速查看攻击集

### 查看总览

```powershell
python .\datasets\github_attack_sets\query_catalog.py summary
```

### 按章节列出样本

```powershell
python .\datasets\github_attack_sets\query_catalog.py list --section 05_indirect_prompt_injection.jsonl --limit 10
```

### 按专项包列出样本

```powershell
python .\datasets\github_attack_sets\query_catalog.py list --pack "MCP / plugin-chain / cross-plugin single-turn" --limit 10
```

### 按关键字筛选

```powershell
python .\datasets\github_attack_sets\query_catalog.py list --keyword mcp --limit 20
```

### 查看单条样本完整内容

```powershell
python .\datasets\github_attack_sets\query_catalog.py show VJL-00001
```

### 导出一批 sample id

```powershell
python .\datasets\github_attack_sets\query_catalog.py export-ids --section 04_protected_paths_skills_plugins.jsonl --output .\tmp_sample_ids.txt
```

## 3. 在项目页面里怎么用

项目启动后：

- 前端：`http://127.0.0.1:5173`
- 默认账号：`admin / admin123`

推荐在“样本执行”页使用攻击集：

1. 打开“样本执行”页
2. 用章节、专项包、风险等级、测试模式筛选样本
3. 勾选一个或多个样本
4. 选择目标 Agent
5. 点击“批量建任务”或“批量执行”
6. 在右侧查看任务状态、事件和报告
7. 需要时点击“下载整批报告”

### 最值得先跑的章节

- `04_protected_paths_skills_plugins.jsonl`
- `05_indirect_prompt_injection.jsonl`
- `06_multi_turn_context_pollution.jsonl`
- `07_output_leakage_and_redaction.jsonl`

### 单轮 / 多轮怎么选

- 无状态或每次请求独立的模型接口：优先用 `single_turn`
- 会保留上下文、会话和工具状态的 Agent：优先用 `multi_turn`

MCP / plugin-chain / cross-plugin 这类场景，优先用：

- `04_mcp_plugin_chain_cross_plugin_single_turn.jsonl`
- `04_mcp_plugin_chain_cross_plugin_multi_turn.jsonl`

## 4. 用 API 怎么接

如果你不想走页面，可以直接走后端接口。

### 先登录

```http
POST /api/auth/login
```

### 查攻击集总览

```http
GET /api/samples/summary
GET /api/samples/sections
GET /api/samples/packs
GET /api/samples?page=1&page_size=20&section=05_indirect_prompt_injection.jsonl
GET /api/samples/{sample_id}
```

### 从样本创建任务

单条样本：

```http
POST /api/attack-tasks/from-sample
```

批量样本：

```http
POST /api/attack-tasks/batch-from-samples
```

### 下载整批报告

```http
POST /api/reports/batch-download
```

## 5. 最低可用流程

如果你只是想确认攻击集真的能用，按这个顺序最稳：

1. `python .\datasets\github_attack_sets\validate_catalog.py`
2. `python .\datasets\github_attack_sets\query_catalog.py summary`
3. 启动平台：`.\start.ps1`
4. 登录前端
5. 在“样本执行”页筛一个章节
6. 多选样本，点击“批量建任务”
7. 确认任务创建成功

如果要真正跑模型，还要补 `.env` 里的 AI Provider 配置。

## 6. 当前边界

- 攻击集本身已经能被页面和 API 读取
- 平台当前能正常做样本筛选、任务创建和报告归档
- 如果 `AI_PROVIDER=disabled`，任务执行会失败，但攻击集本身并没有坏
- 多轮样本是否“跑得像真实 Agent”，取决于你的 Runtime 是否保留会话状态
